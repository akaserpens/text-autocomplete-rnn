# Задача проекта
* Есть входной датасет с короткими текстовыми постами
* Взять датасет, очистить его, подготовить для обучения модели.
* Реализовать и обучить модель для предсказания продолжения фразы на основе рекуррентных нейронных сетей.
* Замерить качество разработанной и обученной модели.
* Взять предобученную модель из Transformers и замерить её качество.
* Проанализировать результаты и дать рекомендации: стоит ли использовать лёгкую модель или лучше постараться поработать с ограничениями по памяти и использовать большую предобученную.

Было реализовано две модели:
* Модель на базе LSTM, предсказывающая один следующий токен по принципу классификатора. В режиме генерации рекурсивно предсказывает по одному токену, пока не будет достигнуто максимальное количество.
* Модель с архитектурой Encoder-Decoder, генерирующая окончание предложения по его началу. Без Attention

# Структура проекта
* /configs - содержит файлы конфигурации
* /data - директория для хранения исходного и промежуточных датасетов
* /models - содержит веса обученных моделей
* /src - исходный код

# Конфигурация
* dataset - настройки датасета
  * source - url исходного файла с данными
  * dir - директория для хранения исходного и промежуточных файлов с данными
  * original_filename - имя файла для исходного датасета. Имена остальных файлов будут формироваться на основе исходного добавлением постфиксов
  * max_examples: количество строк, которое будет взято для обучения модели. Укажите -1, чтобы использовать все строки. Используется на этапе тестирования и отладки для сокращения объема данных
* train - настройки обучения
  * batch_size - размер батча
  * max_epoch - количество эпох обучения
* model - настройки моделей
  * hidden_dim - количество скрытых слоев. Используется для обеих моделей
  * dir - директория для сохранения весов моделей
* clearml - настройки clear ml сервера
  * enabled - если 0, данные не будут отправляться в clear ml
  * api_host - api_host
  * web_host - web_host
  * files_host - web_host

Также для корректной работы ClearML нужно задать переменные окружения:
* CLEARML_KEY - api access key
* CLEARML_SECRET - api secret key

Если в корне проекта лежит файл `.env`, переменные окружения будут загружен из него
